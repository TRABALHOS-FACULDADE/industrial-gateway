## Como contribuir

1. Crie sua branch a partir da `develop`: `git checkout -b feature/nome-da-feature`
2. Faça commits pequenos e descritivos
3. Abra PR para `develop`, nunca para `main`
4. PR precisa de ao menos 1 aprovação antes de mergear
5. Alterações no `docker-compose.yml` e `init-sql.sql` devem ser comunicadas no grupo
